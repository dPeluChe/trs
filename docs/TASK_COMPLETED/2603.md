# 2026-03-18 ‚Äî Phase 1: New Features + Competitor Analysis

## Context
Competitive analysis of RTK (v0.31.0), tokf, ccp, claw-compactor, QMD, and Pi Coding Agent.
Identified gaps vs RTK and implemented Phase 1 features based on findings.
Evaluated and **discarded** MCP Server integration (adds complexity without value for a unidirectional pipe tool).

## Completed

### Emoji stripping (global, default)
- Added `strip_emojis()` to `src/router/handlers/common.rs`
- Applied in `ParseHandler::read_input()` and `Router::process_stdin()`
- Strips decorative emoji (U+1Fxxx ranges + select dingbats like U+274C, U+2705, U+2728)
- Preserves functional symbols: checkmarks, crosses, bullets, warning signs (used by test runners/build tools)
- **Why**: RTK #603 found emojis confuse non-Claude LLMs causing retry loops

### `trs json` command ‚Äî JSON structure without values
- New handler: `src/router/handlers/json.rs`
- Shows keys + types + array lengths, no values
- Supports `--depth N` for max depth limit
- Supports `--json` output for programmatic schema
- Long strings show length: `String[1024]`, homogeneous arrays show type: `Array[42] of Object`
- 7 unit tests

### `gh` (GitHub CLI) parsers
- `trs gh pr list` / `trs parse gh-pr` ‚Äî pull request list
- `trs gh issue list` / `trs parse gh-issue` ‚Äî issue list
- `trs gh run list` / `trs parse gh-run` ‚Äî workflow runs with status detection
- Supports **both** TTY format (emojis + `#`) and non-TTY format (TSV from subprocess capture)
- Auto-detect in classifier + pipe syntax

### Tee system ‚Äî failure output recovery
- On non-zero exit code, saves full stdout+stderr to `~/.trs/tee/<timestamp>_<cmd>.log`
- Prints hint: `[full output: path]` so agent can recover full output
- Auto-cleanup: keeps last 20 files

### Deep competitor analysis (repos cloned via ghq)
- RTK: 64 modules, TOML filters (60+), ParseResult tiers, configurable limits, SQLite tracking, discover command
- tokf: TOML filter pipeline (8 stages), Lua scripting, eject system, verify framework, color passthrough
- claw-compactor: Ionizer (JSON sampling), LogCrunch (log folding), DiffCrunch, Neurosyntax (AST regex fallback), SemanticDedup (shingles)
- ccp: Conservative philosophy, YAML filters, command mutation (`--porcelain`), passthrough for structured output

## Decisions

### Discarded: MCP Server
trs is a unidirectional pipe/filter. MCP adds protocol overhead without reducing more tokens.

### Discarded: Cloning QMD/Pi repos (later cloned RTK/tokf/ccp/claw-compactor instead)
TypeScript repos not directly reusable. Only patterns extracted.

### Discarded (for now): TOML filter plugin system
tokf proves it works but adds significant complexity. Consider for Phase 3 when user demand exists.

### Discarded (for now): Lua scripting escape hatch
Overengineering for current stage. Revisit if TOML filters are adopted.
